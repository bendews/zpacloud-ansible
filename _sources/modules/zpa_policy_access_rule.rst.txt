zpa_policy_access_rule -- Create a Policy Access Rule
=====================================================

.. contents::
   :local:
   :depth: 1


Synopsis
--------

This module create/update/delete a Policy Access Rule in the ZPA Cloud.







Parameters
----------

  policy_set_id (type: str)
    


  action (type: str)
    This is for providing the rule action.


  action_id (type: str)
    This field defines the description of the server.


  priority (type: str)
    


  id (type: str)
    


  policy_type (type: str)
    


  rule_order (type: str)
    


  default_rule (type: bool)
    This is for providing a customer message for the user.


  operator (type: str)
    This denotes the operation type.


  app_connector_groups (type: list)
    List of the app connector group IDs.


    id (type: str)
      List of the app connector group IDs.



  app_server_groups (type: list)
    List of the server group IDs.


    id (type: str)
      List of the server group IDs.



  custom_msg (type: str)
    This is for providing a customer message for the user.


  lss_default_rule (type: bool)
    


  name (required, type: str)
    This is the name of the policy.


  conditions (type: list)
    This is for proviidng the set of conditions for the policy.


    negated (type: bool)
      


    operator (type: str)
      This denotes the operation type.


      operands (type: list)
        This signifies the various policy criterias.


        idp_id (type: str)
          


        lhs (type: str)
          This signifies the key for the object type.


        name (type: str)
          This signifies the key for the object type.


        rhs (type: str)
          This denotes the value for the given object type. Its value depends upon the key.


        object_type (type: str)
          This is for specifying the policy criteria.












Examples
--------

.. code-block:: yaml+jinja

    
    - name: Access Policy - Intranet Web Apps
      willguibr.zpacloud.zpa_policy_access_rule:
        name: "Intranet Web Apps"
        description: "Intranet Web Apps"
        action: "ALLOW"
        rule_order: 1
        operator: "AND"
        conditions:
          - negated: false
            operator: "OR"
            operands:
              - name: "app_seg_intranet"
                object_type: "APP"
                lhs: "id"
                rhs: "{{ app_seg_intranet.data.id }}"
          - negated: false
            operator: "OR"
            operands:
              - name: "sg_seg_intranet"
                object_type: "APP_GROUP"
                lhs: "id"
                rhs: "{{ seg_intranet.data.id }}"
          - negated: false
            operator: "OR"
            operands:
              - name: "engineering_group"
                object_type: "SCIM_GROUP"
                lhs: "{{ user_okta.data[0].id }}"
                rhs: "{{ engineering_group.data[0].id }}"
