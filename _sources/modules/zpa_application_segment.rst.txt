.. _zpa_application_segment_module:


zpa_application_segment -- Create an application segment in the ZPA Cloud.
==========================================================================

.. contents::
   :local:
   :depth: 1


Synopsis
--------

This module will create/update/delete an application segment







Parameters
----------

  name (required, type: str)
    Name of the application.


  description (type: str)
    Description of the application.


  default_max_age (type: str)
    default_max_age


  ip_anchored (type: bool)
    Whether Source IP Anchoring for use with ZIA, is enabled or disabled for the app.


  tcp_port_range (required, type: list)
    List of tcp port range pairs, e.g. [22, 22] for port 22-22, [80, 100] for 80-100.


    from (type: str)
      List of valid TCP ports. The application segment API supports multiple TCP and UDP port ranges.


    to (type: str)
      List of valid TCP ports. The application segment API supports multiple TCP and UDP port ranges.



  udp_port_range (required, type: list)
    List of udp port range pairs, e.g. [‘35000’, ‘35000’] for port 35000.


    from (type: str)
      List of valid UDP ports. The application segment API supports multiple TCP and UDP port ranges.


    to (type: str)
      List of valid UDP ports. The application segment API supports multiple TCP and UDP port ranges.



  double_encrypt (type: bool)
    Whether Double Encryption is enabled or disabled for the app.


  icmp_access_type (type: str, default: NONE)
    icmp access type.


  default_idle_timeout (type: str)
    default idle timeout.


  passive_health_enabled (type: bool)
    passive health enabled.


  bypass_type (type: str, default: NEVER)
    Indicates whether users can bypass ZPA to access applications.


  is_cname_enabled (type: bool)
    Indicates if the Zscaler Client Connector (formerly Zscaler App or Z App) receives CNAME DNS records from the connectors.


  config_space (type: str, default: DEFAULT)
    config space.


  health_reporting (type: str, default: NONE)
    Whether health reporting for the app is Continuous or On Access. Supported values are NONE, ON_ACCESS, CONTINUOUS


  log_features (type: str)
    log features.


  server_groups (required, type: list)
    ID of the server group.


    id (required, type: str)
      ID of the server group.



  segment_group_id (required, type: str)
    ID of the segment group.


  segment_group_name (type: str)
    segment group name.


  health_check_type (type: str)
    health check type.


  enabled (type: bool)
    Whether this application is enabled or not.


  domain_names (required, type: list)
    List of domains and IPs.









Examples
--------

.. code-block:: yaml+jinja

    
    - name: Create/Update/Delete an application segment.
      willguibr.zpacloud.zpa_application_segment:
        name: Example Application Segment
        description: Example Application Segment
        enabled: true
        health_reporting: ON_ACCESS
        bypass_type: NEVER
        is_cname_enabled: true
        tcp_port_range:
          - from: "80"
            to: "80"
        domain_names:
          - crm.example.com
        segment_group_id: "216196257331291896"
        server_groups:
          - "216196257331291969"
