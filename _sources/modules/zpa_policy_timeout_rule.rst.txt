zpa_policy_timeout_rule -- Create a Policy Timeout Rule
=======================================================

.. contents::
   :local:
   :depth: 1


Synopsis
--------

This module create/update/delete a Policy Timeout Rule in the ZPA Cloud.







Parameters
----------

  client_id (type: str)
    


  client_secret (type: str)
    


  customer_id (type: str)
    


  action (type: str)
    This is for providing the rule action.


  action_id (type: str)
    This field defines the description of the server.


  priority (type: str)
    


  reauth_default_rule (type: bool)
    


  id (type: str)
    


  policy_type (type: str)
    


  rule_order (type: str)
    


  default_rule (type: bool)
    This is for providing a customer message for the user.


  operator (type: str)
    This denotes the operation type.


  description (type: str)
    This is the description of the access policy.


  custom_msg (type: str)
    This is for providing a customer message for the user.


  name (required, type: str)
    This is the name of the timeout policy.


  default_rule_name (type: str)
    


  reauth_idle_timeout (type: str)
    


  reauth_timeout (type: str)
    


  conditions (type: list)
    


    id (type: str)
      


    negated (type: bool)
      


    operator (required, type: str)
      


    operands (type: list)
      


      id (type: str)
        


      idp_id (type: str)
        


      name (type: str)
        


      lhs (required, type: str)
        


      rhs (type: str)
        


      rhs_list (type: list)
        


      object_type (required, type: str)
        




  state (type: str, default: present)
    Whether the app should be present or absent.
